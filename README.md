### Το bug που διορθώθηκε

Το παλιό `daily_sync.py` έγραφε στο Sheet **ευρώ** (`1547.73`), ενώ τα
`data_sync.py` και `sales_sync.py` έγραφαν **λεπτά** (`154773`).

Η εφαρμογή διαβάζει πάντα `/100`. Άρα ό,τι έγραφε το `daily_sync.py` εμφανιζόταν
100 φορές μικρότερο: **1.547,73 € → 15,48 €**.

Τώρα και τα τρία περνούν από το `core/sheets.py`, που κάνει τη μετατροπή μία
φορά, σωστά.

## Γιατί δεν σκάει πια

Το παλιό app έπεφτε με segmentation fault στο Streamlit Cloud. Τρεις αιτίες,
τρεις λύσεις:

**1. `.style.format()`**
Ο pandas Styler χτίζει ολόκληρο HTML στη μνήμη. Με 10.000 γραμμές παραστατικών,
τέλος. → Μορφοποιούμε **πριν** το dataframe, σε απλά strings.

**2. Εκατοντάδες widgets**
Το παλιό loop έφτιαχνε `st.text_input` σε κάθε γραμμή της σελίδας «Μήνας».
→ **Ένα** `data_editor`, όσες γραμμές κι αν έχει.

**3. Uncached client**
Κάθε rerun έφτιαχνε νέο `gspread` connection. → `@st.cache_resource` singleton.
Στο cache μπαίνουν **μόνο** DataFrames — ποτέ ζωντανά worksheet objects.

**Και το OCR δεν φορτώνεται ποτέ στο Streamlit Cloud.** Το `pdf2image` και το
`pytesseract` χρειάζονται system libs (poppler, tesseract) που δεν υπάρχουν εκεί
— και μόνο το import τους ρίχνει την εφαρμογή. Γι' αυτό υπάρχουν δύο αρχεία:

- `requirements.txt` → Streamlit Cloud. **Χωρίς OCR.**
- `requirements-ocr.txt` → GitHub Actions. Εκεί εγκαθιστούμε tesseract & poppler.

## Πωλήσεις — κάθε 10 λεπτά από τις 18:00

Το `sales_sync.yml` τρέχει **κάθε 10 λεπτά, 18:00 → 02:00** (ώρα Ελλάδας).
Μόλις έρθει η αναφορά της ημέρας, εμφανίζεται στην Επισκόπηση μέσα σε λεπτά.

### Γιατί είναι ασφαλές να τρέχει τόσο συχνά

Το OCR είναι **αργό** — 20-30 δευτερόλεπτα ανά PDF. Με 54 εκτελέσεις τη νύχτα,
χωρίς προστασία θα έκανε OCR στα ίδια PDF ξανά και ξανά:

```
54 εκτελέσεις × 30" = 27 λεπτά OCR κάθε νύχτα, τζάμπα
```

**Η λύση:** το `sales_sync.py` περνάει τις **ήδη γνωστές μέρες** στο
`fetch_sales`. Αν η μέρα του email είναι ήδη στο Sheet, το PDF **δεν ανοίγει καν**.

```
ΠΡΙΝ:  54 × 30"  = 27 λεπτά OCR
ΤΩΡΑ:   1 × 30"  = 30 δευτερόλεπτα
```

Ο έλεγχος γίνεται **πριν** το OCR, όχι μετά.

### Θερινή/χειμερινή ώρα

Το GitHub Actions τρέχει πάντα σε **UTC** και δεν ξέρει από θερινή ώρα:

| | 18:00 Ελλάδας |
|---|---|
| Χειμώνας (UTC+2) | 16:00 UTC |
| Καλοκαίρι (UTC+3) | 15:00 UTC |

Το cron καλύπτει **15:00 → 00:59 UTC**, δηλαδή και τις δύο περιπτώσεις.
Πρακτικά: το καλοκαίρι ξεκινάει 18:00, τον χειμώνα 17:00. Μια ώρα νωρίτερα
δεν βλάπτει.

## Ο καθαρισμός δεν κρύβεται πια

**Το πρόβλημα:** ο καθαρισμός ήταν μέσα σε ένα κλειστό expander, σε μια από τρεις
καρτέλες. Ο χρήστης έβλεπε το πρόβλημα («157 γραμμές περισσεύουν») αλλά όχι τη λύση.

**Τώρα:**

1. **Προειδοποίηση στην κορυφή** της σελίδας Παραστατικά, έξω από κάθε expander:
   ```
   ⚠️ 157 από τις 235 γραμμές δεν έχουν αριθμό παραστατικού (67%).
      Τα σύνολα παραπάνω μπορεί να είναι φουσκωμένα.
   ```

2. **Το expander ανοίγει μόνο του** όταν υπάρχει πρόβλημα.

3. **Ο καθαρισμός δείχνει τι θα διορθώσει** πριν καν πατήσεις — μέτρημα από το
   cache, χωρίς κλήσεις δικτύου.

4. **Η επαλήθευση έχει κουμπί** που πάει κατευθείαν στον καθαρισμό.

Ένα εργαλείο που βρίσκει το πρόβλημα αλλά δεν το λύνει είναι μισό εργαλείο.

## 🔄 Χειροκίνητος συγχρονισμός πωλήσεων

### Το πρόβλημα

Το Streamlit Cloud **δεν έχει tesseract/poppler** — άρα το OCR δεν τρέχει εκεί.
Το παλιό κουμπί «Πωλήσεις» έλεγε «το OCR δεν είναι διαθέσιμο» και σταματούσε.

**Άχρηστο:** όταν ο αυτόματος συγχρονισμός αποτύχει, δεν είχες κανέναν τρόπο να
το διορθώσεις από την εφαρμογή.

### Η λύση

Το κουμπί τώρα **ξεκινάει το GitHub Action** — εκεί το OCR δουλεύει.

Χρειάζεται ένα **Personal Access Token**:

**1. Φτιάξε το token**

GitHub → Settings → Developer settings → Personal access tokens →
**Fine-grained tokens** → Generate new token

| Πεδίο | Τιμή |
|---|---|
| Repository access | Only select repositories → `ab-skyros-dashboard` |
| Permissions → **Actions** | **Read and write** |
| Expiration | 1 year |

**2. Βάλ' το στα Streamlit secrets**

```toml
GITHUB_TOKEN = "github_pat_..."
GITHUB_REPO  = "abskyros/ab-skyros-dashboard"
```

**Αν λείπουν**, το κουμπί σου λέει να πας στο GitHub. Δεν σκάει.

### Επιπλέον: η κατάσταση του συγχρονισμού

Με το token, η εφαρμογή δείχνει και **αν ο τελευταίος αυτόματος συγχρονισμός
απέτυχε**:

```
⚠️ Ο τελευταίος αυτόματος συγχρονισμός πωλήσεων ΑΠΕΤΥΧΕ.
   Γι' αυτό δεν ενημερώνονται οι πωλήσεις.
   [Δες το σφάλμα στο GitHub]
```

Ένα σιωπηλό σφάλμα είναι χειρότερο από ένα θορυβώδες.

## Φρεσκάδα δεδομένων

Η Επισκόπηση δείχνει τώρα **πόσο πίσω είναι** οι πωλήσεις:

| Διαφορά | Τι λέει |

## ⏰ Πωλήσεις — από τις 21:00, κάθε 10 λεπτά

Η αναφορά πωλήσεων βγαίνει **μετά τις 21:00 ώρα Ελλάδας**. Το job ελέγχει κάθε
10 λεπτά ώσπου να τη βρει.

### Το πρόβλημα της ώρας

Το GitHub cron τρέχει **πάντα σε UTC** και **δεν ξέρει από θερινή ώρα**:

| | 21:00 Ελλάδας |
|---|---|
| Καλοκαίρι (EEST, UTC+3) | **18:00 UTC** |
| Χειμώνας (EET, UTC+2) | **19:00 UTC** |

Ένα σταθερό cron θα ήταν **λάθος τη μισή χρονιά**.

### Η λύση

**Το cron τρέχει ευρέως** (18:00–03:59 UTC = καλύπτει και τις δύο περιπτώσεις),
**και το script ελέγχει την πραγματική ώρα Ελλάδας**:

```python
now = now_greece()           # zoneinfo — ξέρει από θερινή ώρα
if now.hour < 21:
    return                   # πολύ νωρίς, βγαίνω
```

Ο έλεγχος γίνεται **εκεί που υπάρχει η γνώση**. Η Python ξέρει από `zoneinfo`,
το cron όχι.

### Ποια μέρα ψάχνουμε

Η αναφορά αφορά **τη μέρα που πέρασε**:

```
Δευτέρα 23:00  →  ψάχνουμε τη ΔΕΥΤΕΡΑ
Τρίτη   01:00  →  ψάχνουμε ΑΚΟΜΑ τη ΔΕΥΤΕΡΑ
```

Αν το μπερδεύαμε, θα λέγαμε «όλα εντάξει» ενώ λείπει η χθεσινή.

### Το κόστος — και πώς μειώθηκε 18×

Το OCR χρειάζεται `apt-get install tesseract poppler` — **40 δευτερόλεπτα** σε
κάθε εκτέλεση.

Με 54 εκτελέσεις τη νύχτα: **36 λεπτά runner time**, κάθε βράδυ, τζάμπα.

**Λύση:** το `sales_precheck.py` ελέγχει σε **2 δευτερόλεπτα** αν αξίζει —
χωρίς να εγκαταστήσει τίποτα. Το workflow το ρωτάει πρώτα:

```yaml
- name: Ελαφρύς έλεγχος
  id: check
  run: |
    if python jobs/sales_precheck.py; then
      echo "work=yes" >> $GITHUB_OUTPUT
    fi

- name: OCR
  if: steps.check.outputs.work == 'yes'    # ← μόνο αν χρειάζεται
  run: sudo apt-get install -y tesseract-ocr poppler-utils
```

| | Εκτελέσεις | Χρόνος |
|---|---|---|
| **Πριν** | 54 × 40" | **36 λεπτά** |
| **Τώρα** | 1 × 40" + 53 × 2" | **2 λεπτά** |

Μόνο **μία** εκτέλεση εγκαθιστά OCR — αυτή που βρίσκει την αναφορά.
